Hangman: Candy Edition

Hangman: Candy Edition

Project Description:

This application is a candy-themed hangman game. A word is chosen randomly from an array of candy-themed words and displayed to the user with a __ representing each letter. The user starts the game with 12 lifepoints and they lose a lifepoint every time they guess a letter. They guess letters by pressing the key on the keyboard. If it is a correct guess, it is displayed in the correct location in the word. If it is an incorrect guess, it is added to the wrong guesses section. The user wins if they guess the word before they run out of lifepoints. The application keeps track of their wins and chooses a new word. If they lose, the game resets.

Technology Used:

  • HTML
  • CSS
  • JavaScript

Crystal Collector Game

Crystal Collector Game

Project Description:

This application is a game in which a random integer is chosen between 19 and 120. This is the target number the user is trying to match. Then there are four crystals, each with a random, hidden value between 1 and 12. When the user clicks on a crystal, the value of that crystal is added to their total. The goal of the game is to get their total to match the target number. If their total exceeds the target number, they lose and new values are chosen for the crystals and target number.

RouteVisor

RouteVisor

Project Description:

This application splits a long road trip into shorter legs based on the user's input of number of miles they want to travel in one stretch. Dog-Go

Dog-Go

Project Description:

This application allows users to create a profile and book appointments with dog walkers to provide their pet with the exercise they need. People can also sign up to become a dog walker on the site.
Users choose at the home screen if they want to "Schedule a Walk" or "Walk a Dog". Based on their selection, they are routed to either a customer information form or walker information form. Walkers are added to a walker table in the database and shown all walkers in the database when they sign up. Customers are added to the customer table and routed to a calendar to book their appointment.
When the customer chooses a date on the calendar, they are shown available half hour appointments from 9:00am to 5:00pm on that day. When they choose one of the available appointments, it is added to an appointments table and no longer shown when the date is clicked.

Technology Used:

  • MySQL
  • Node
  • Handlebars
  • Express
  • Sequelize
  • Heroku
  • Moment.js
  • LESS

New York Times Scraper

New York Times Scraper

Project Description:

This application uses cheerio and axios node packages to scrape the New York Times website and store headlines and links to articles in a MongoDB database. Then an ajax get request is used to pull the headlines and links from the database and display them to the user on the front end.

Users can then either go to the article through the link or click on the headline to make a note on the article. When they click on a headline, a modal appears with a form for them to submit a note on the selected article. On submission, the note is stored in that article document in the database.

Technology Used:

  • MongoDB/Mongoose
  • Node
  • Express
  • Cheerio
  • Heroku
